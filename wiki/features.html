功能說明

以下內容以 4.1.3 的行為為準。

1. 匹配流程（Queue + Confirm GUI）

  1. 玩家從匹配面板進入隊列。
  2. 關閉 GUI 後，ActionBar 仍會持續顯示搜尋秒數與等待人數。
  3. 配對成功後，優先用 GUI 提供接受/拒絕。
  4. 若 GUI 不可用，會自動回退到 /dt accept / /dt decline。

2. 隊列看門狗（Queue Watchdog）

  • 定期清理離線玩家的卡住隊列資料。
  • 定期清理目標競技場無效的等待資料。
  • 定時觸發等待中的競技場做匹配檢查。

可用 /dt doctor queue 或 /dt queue debug 觀察運作狀態。

3. 離場懲罰（可配置）

主動 /quit 或斷線判負才套用 死亡不套用
  • 可設定是否扣點。
  • 可設定扣點數值。
  • 可設定是否套用隊列冷卻與冷卻秒數。

4. 連勝/連敗（Streak）

  • 每場結束後更新 streak。
  • 可顯示當前 streak。
  • 可設定平手時是否重置。

5. 玩家自訂隊列音效

玩家可用 /dt queue sound on|off|toggle|status 控制自己的排隊提醒音效。

管理員可透過設定限制玩家是否能自行切換。

6. 戰報匯出（管理員）

指令：/dt record export [player] [json|csv] [limit]

預設 limit=100，最大 limit=1000。

7. 重啟保護

  • 伺服器重啟時，戰鬥中玩家會收到 Title + 聊天提示。
  • 重啟期間離線不套用中途離場懲罰。
