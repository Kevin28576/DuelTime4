疑難排解

常見錯誤與建議處理方式。

1) 啟動後大量 ClassNotFoundException

例如：net.minecraft.server.craftbukkit.Packet

  • 多半是舊版 NMS 路徑在新 Paper 已變更。
  • 先確認使用最新 DuelTime4。
  • 避免在自訂功能中硬寫舊版 NMS class 名稱。

2) Maven 編譯錯誤：illegal character '\ufeff'

  • 有 Java 檔含 BOM（Byte Order Mark）。
  • 把該檔另存成 UTF-8（無 BOM）。
  • IDE 預設儲存格式改為 UTF-8 without BOM。

3) YAML 無法載入（特殊字元 / code point）

  • 語言檔可能貼入了不可見字元。
  • 確認縮排使用空白，不要用 Tab。
  • 含 : 的字串建議加引號。

4) 更新器抓不到新版本

優先檢查：

Updater.manifest-url
Updater.fallback-version-url
Updater.download-url-template

GitHub release 路徑要是 v{version}，例如 v4.1.3。

5) 隊列卡住 / 配不到對手

  1. 執行 /dt doctor queue 確認 watchdog 是否啟用。
  2. 執行 /dt queue debug 檢查 waiting 與 pending。
  3. 確認競技場狀態是 WAITING 且可加入。
  4. 確認玩家沒有處於離場懲罰冷卻中。

6) 快速自檢

Java 21 Paper 1.21+ config.yml 縮排正確 語言檔無亂碼與控制字元
